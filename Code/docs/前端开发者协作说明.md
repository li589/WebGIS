# WebGIS 前端开发者协作说明

这份文档给负责前端开发的同学使用。

当前项目约定：

- 仓库地址：`https://github.com/li589/WebGIS`
- 稳定分支：`main`
- 日常开发分支：`dev`
- 你主要负责目录：`Code/frontend`
- 对方主要负责目录：`Code/backend`、`Code/algorithms`
- 双方共同维护目录：`Code/docs`、`Code/shared`、`README.md`

说明：早期约定中的 `Code/infra` 目录当前不存在；Redis / MinIO 配置在 `Code/backend/docker-compose.yml`，一键启动在仓库根 `launch.py`。

## 第一次获取项目

第一次在本地使用这个项目时：

```bash
git clone https://github.com/li589/WebGIS.git
cd WebGIS
git checkout dev
git pull origin dev
```

## 你平时主要改哪里

你主要修改：

- `Code/frontend`

尽量不要主动改这些目录，除非已经和后端同学确认：

- `Code/backend`
- `Code/algorithms`

共同文件如果要改，最好先说一声：

- `Code/docs`
- `Code/shared`
- `README.md`

## 每次开始开发前怎么同步

由于你主要做前端，不希望覆盖自己正在开发的前端代码，所以推荐先按目录同步后端和公共文件，而不是直接整分支 `pull`。

建议执行：

```bash
git checkout dev
git fetch origin
git restore --source origin/dev -- Code/backend Code/algorithms
git restore --source origin/dev -- Code/shared Code/docs README.md Code/README.md
```

这样做的效果是：

- 更新后端代码
- 更新共享协议和文档
- 尽量不动你自己的 `Code/frontend`

## 如果你当前前端还没提交

在同步前先检查状态：

```bash
git status
```

如果你本地还有重要前端改动，建议先做其中一种：

### 方案 1：直接提交

```bash
git add .
git commit -m "wip(frontend): current progress"
```

### 方案 2：临时暂存

```bash
git stash
```

恢复时：

```bash
git stash pop
```

## 你日常开发和提交的推荐流程

```bash
git checkout dev
git fetch origin
git restore --source origin/dev -- Code/backend Code/algorithms
git restore --source origin/dev -- Code/shared Code/docs README.md Code/README.md
```

然后在 `Code/frontend` 中开发。

开发完成后：

```bash
git add .
git commit -m "feat(frontend): 简要说明本次改动"
git push origin dev
```

提交信息示例：

```bash
git commit -m "feat(frontend): add OSM base map"
git commit -m "fix(frontend): adjust floating panels"
git commit -m "refactor(frontend): split dashboard layout"
```

## 更推荐的方式

如果你有一块独立前端功能要开发，推荐从 `dev` 拉自己的功能分支：

```bash
git checkout dev
git pull origin dev
git checkout -b feature/frontend-map-panel
```

开发完成后：

```bash
git add .
git commit -m "feat(frontend): complete map panel"
git push -u origin feature/frontend-map-panel
```

然后再由项目负责人合并到 `dev`。

## 什么时候用正常 `git pull`

如果后续你开始同时频繁改这些公共目录：

- `Code/shared`
- `Code/docs`
- 根目录说明文档

那就不建议只按目录同步了，应改成：

```bash
git checkout dev
git pull origin dev
```

## 前端开发时特别要注意

- 提交前先 `git status`
- 尽量只提交与你当前前端功能相关的文件
- 不要把 `node_modules`、构建产物、缓存提交到仓库
- 如果需要改共享协议，先和后端同学确认字段结构

## 常用命令速查

切到 `dev`：

```bash
git checkout dev
```

同步后端代码：

```bash
git fetch origin
git restore --source origin/dev -- Code/backend
```

同步公共文档和共享协议：

```bash
git fetch origin
git restore --source origin/dev -- Code/shared Code/docs README.md Code/README.md
```

查看当前状态：

```bash
git status
```

提交并推送：

```bash
git add .
git commit -m "feat(frontend): 本次改动说明"
git push origin dev
```

## 一句话原则

你负责前端，就重点维护 `Code/frontend`；同步时优先拉后端和共享目录，不要随手覆盖自己正在做的前端代码。
