# 课题组数据全链路（代码事实同步 · 2026-07-21）

本文记录「下载 → 计算 → 展示 + 工作流模块扩展」落地后的**当前有效事实**，供协作与回归对照。  
设计决策见 Cursor plan `research_data_fullchain`；本文不替代各 README，只同步实现状态。

## 1. 画布执行主链（Phase 1）

| 项 | 事实 |
|----|------|
| 编译器 | `backend/app/services/workflow_graph_compiler.py`：LiteGraph `{nodes,links,properties}` → Python `workflow_definition` |
| API | `POST /workflow-definitions/compile`（需写权限） |
| 前端 Run | `WorkflowEditorPanel` 序列化画布 → `compileWorkflowGraph` → `runWorkflowForCatalog({ algorithmRequest: { workflow_definition } })` |
| Resolver | 请求已带 `workflow_definition` / `workflow_name` 时，**不**用 catalog `module_name` 覆盖 |
| Catalog 快捷跑 | 侧栏一键跑仍走原路径（无画布定义） |
| 引擎限制 | 本轮编译仅允许 `common` + `python_provider`；天气 / GEE 节点会编译失败并提示 |

验收：改画布节点后 Run，`jobLayer` 对应该编译图，而非静默旧 catalog 定义。

## 2. 可执行数据节点（Phase 2）

节点分类：**数据获取与解析**（`node_template_registry.py`）。

| LiteGraph type | module_name | 说明 |
|----------------|-------------|------|
| `download/remote_fetch` | `remote_fetch` | smb/sftp/ftp/http/https/gs/local → materialize；`?cred=` / `cred_profile` |
| `download/http_open_data` | `http_open_data` | NOAA/NASA/ESA 预设 base + relative_path |
| `archive/extract` | `archive_extract` | zip/tar/gz/tgz |
| `config/read` | `config_read` | JSON/YAML/INI/XML → dict |
| `extract/variable` | `variable_extract` | `UniversalDataReader.read_variable` |
| `format/convert` | `format_convert` | 格式转换；别名 `preprocess_format_convert`（已出 stub 列表） |

实现文件：

- `algorithms/providers/Python/modules/data_access_nodes.py`
- `algorithms/providers/Python/modules/graph_io.py`（`data_source`、`output_map_layer` 等）

开放数据预设默认值 + 设置页 DB 覆盖；bridge 提交时注入 `datasource_selection.open_data_presets`。

## 3. 静态缓存（Phase 3）

| 项 | 事实 |
|----|------|
| 目录 | `BACKEND_STATIC_CACHE_ROOT` 或 `{workspace}/data_access/materialized` |
| TTL | `BACKEND_STATIC_CACHE_TTL_SECONDS`（默认 `0` = 不过期）；`CacheStore` 按 mtime 判定 |
| API | `GET /config/data-cache/overview`、`POST /config/data-cache/evict` |
| 数据根扫描 | overview / data-source 配置中返回 `discovered_datasets`（`BACKEND_DATA_ROOT` 一级目录） |
| 天气网格 | **不混用**本策略，仍走短 TTL |

## 4. 设置面（Phase 4）

`DataSourceSettings.vue`：

1. 数据根 / 输出根 / 已发现数据集  
2. 开放数据预设（JSON 可编辑）  
3. 图层 URI 覆盖（**嵌套** JSON，DB **优先于** `BACKEND_REMOTE_LAYER_DATA_URIS`）  
4. 静态缓存占用与清理  

`RemoteStorageSettings.vue`：说明工作流「远程拉取」可用 `cred_profile` / `?cred=`。

持久化：`research_data_settings.sqlite3`（与 GEE credentials 同目录）。

## 8. 快速用法

1. 设置 → 数据源：确认 `BACKEND_DATA_ROOT`、开放数据预设、缓存目录  
2. 设置 → 远程存储：配置 NAS profile  
3. 工作流编辑器：拖「数据源 / 远程拉取 / 解压 / 提取变量 / 算法模块」并连线  
4. 关联 python_provider 图层 → Run → 状态面板看进度
