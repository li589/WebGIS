# 远程存储接入说明

最后更新：2026-07-17

## 能力概览

算法取数（`data_access`）与下载链（`source_fetcher`）共用 [`Code/shared/remote_sources/`](../Code/shared/remote_sources/)：

| Scheme | Transport | 备注 |
|--------|-----------|------|
| `sftp://` | paramiko | SSH 仅 SFTP，无远程命令 |
| `smb://` | smbprotocol | `smb://host/share/path` |
| `ftp://` / `ftps://` | ftplib | 明文 FTP 默认关闭 |
| `gs://` | google-cloud-storage | 服务账号 JSON 存入凭证 secret |

## 配置

1. 设置页 → **远程存储**，或 `PUT /config/remote-storage/{profile_id}`（需 `X-API-Key`）
2. URI 绑定凭证：`sftp://nas/data/a.tif?cred=nas-lab`
3. 也可按 host+protocol 匹配默认 profile（启用态、最近更新优先）
4. SMB profile 必须配置 `extra.default_share`，否则连通性测试无法构造 probe URI
5. SFTP 内网可设 `extra.host_key_policy=auto_add`（设置页勾选）

凭证库：`workflow_state/remote_storage_credentials.sqlite3`，加密 key 与 GEE 相同（`BACKEND_GEE_CREDENTIALS_ENCRYPTION_KEY`）。

### 多机 / Worker

- 凭证 **不**随任务载荷下发；Worker 进程需能打开同一凭证库路径并使用同一加密 key
- 推荐：把 `BACKEND_GEE_CREDENTIALS_DB_PATH` 的父目录放到共享盘，或在每台 Worker 上重建相同 profile 后 `POST .../test` 验证
- API 与 Worker 都要能路由到 NAS（防火墙 / VLAN）

## 图层 / 下载用法

- `default_data_access_sources` 候选可直接写完整远端 URI
- 运维覆盖（推荐，不改坏本地已跑通路径）：环境变量 `BACKEND_REMOTE_LAYER_DATA_URIS`（JSON）

```json
{
  "smap-soil": {
    "SMAP_SPL3SMP_E": [
      "smb://192.168.1.10/Geograph/SMAP/SMAP_L3_SM_P_20220101.h5?cred=nas-lab"
    ]
  }
}
```

注入后远端 URI 排在候选最前；本地逻辑名 / 相对路径仍保留作回退。

- 就绪检查：URI 合法 + 凭证可解析；可选 `BACKEND_REMOTE_READINESS_PROBE=true` 做短超时连通性 probe（**不**因对象缺失而 blocked）
- **materialize 落盘后**才在资源 metadata 标 `materialization_status=ready`（任务执行期）；与图层 `run_readiness` 不是同一概念
- 单文件下载上限：`BACKEND_REMOTE_MAX_BYTES`（默认 512 MiB）；大产品务必调高

## 设置页测试

| 模式 | 行为 |
|------|------|
| URI 留空 | 连通性 probe（SMB → share 根；SFTP → 主机根） |
| 填写对象 URI | `stat` 该路径（e2e 前推荐） |

## 密钥历史与本机偏好（2026-07-17）

- **服务端历史**：底图 / `backend_auth` 在 `api_keys.sqlite3` 的 `api_key_history` 表中加密归档；远程存储密码/私钥在 `remote_storage_secret_history`。每次轮换若明文变化则归档，默认每条目保留 20 条（`BACKEND_API_KEY_HISTORY_LIMIT` / `BACKEND_REMOTE_STORAGE_HISTORY_LIMIT`）。
- **API**：`GET|DELETE /config/api-keys/{name}/history`、`POST .../history/{id}/restore`；远程存储同路径前缀 `/config/remote-storage/{id}/history*`。需写鉴权。
- **本机偏好**：浏览器 `localStorage` 保存写请求密钥（`cgda.backend_write_api_key`）、历史折叠/备注、设置默认 Tab。设置页可「清除本机偏好」；**不会**删除服务端历史。
- **GEE**：服务账户 JSON 体积大，不默认多副本存历史；轮换请重新粘贴 JSON（指纹/元数据级审计可后续扩展）。

详见设置页「API管理 / 远程存储」提示文案。
