# 开放门户工具流组件说明

面向课题组与业务同事：说明「数据获取与解析」节点各自做什么、怎么接线，以及系统预设工作流的目的。

---

## 1. 推荐总链路

```text
门户数据下载 →（可选）解压归档 → 提取变量 → 格式转换 →（可选）输出地图图层
```

| 场景 | 用哪条路 |
|------|----------|
| NOAA / NASA / NSIDC / 欧空局 **已知路径** 下载 | **门户数据下载**（`http_open_data`） |
| 实验室 NAS / 任意 URI | **远程拉取**（`remote_fetch`） |
| 本机已有硬盘目录 | **数据源**（`data/source`）参数节点 |

---

## 2. 组件一览

| 界面名称 | 节点类型 | 一句话目的 | 何时用 |
|----------|----------|------------|--------|
| 门户数据下载 | `download/http_open_data` | 按门户预设 + 相对路径下载到缓存 | 开放门户；不负责搜索产品 |
| 远程拉取 | `download/remote_fetch` | 任意 URI 物化到缓存 | NAS、自定义 HTTPS、本地 file |
| 解压归档 | `archive/extract` | 解开 zip/tar/gz；可过滤成员、识别 SAFE | 下载结果是压缩包时 |
| 读取配置 | `config/read` | 读 JSON/YAML/INI/XML 为字典 | 清单、参数文件 |
| 提取变量 | `extract/variable` | 从科学文件抽变量（可裁剪） | HDF5/NetCDF/GeoTIFF/MAT/GRIB |
| 格式转换 | `format/convert` | 转为 mat/npy/geotiff/csv/json | 对接课题组后续算法 |
| 数据源 | `data/source` | 打包本地路径/URI 与数据集键 | 本地盘流水线入口 |
| 输出地图图层 | `output/map_layer` | 把结果登记为可上图产物 | 需要挂到地图时 |

**门户数据下载**关键参数：

- `preset`：`noaa_nomads` / `noaa_goes` / `nasa_earthdata` / `nasa_cmr`（检索预留）/ `nsidc_data` / `esa_copernicus`（目录）/ `esa_download`（下载 CDN）
- `relative_path`：**必填**，具体文件或对象键（请向数据管理员要真实路径）
- `cred_profile`：`earthdata` / `nsidc` / `copernicus`（在「系统设置 → 数据源」配置）
- `force_refresh`：忽略缓存强制重下

**解压归档**增强：

- `member_glob`：如 `*.h5`、`*.nc`
- `recurse_once`：内层再解一层 zip/gz
- 若出现 `*.SAFE` 目录，输出路径会指向 SAFE 根
- **不支持** 7z / rar

**缓存**：下载写入静态 materialize 目录；带鉴权头时缓存键会区分凭证；支持 ETag 条件请求。可在设置页查看/清理缓存。

## 3. 系统预设工作流（目的说明）

| 模板 ID | 目的 | 前置 | 你需要改的 |
|---------|------|------|------------|
| `open_data_noaa_grib_sample` | 演示 NOAA 下载→提取→转换 | 通常无需凭证 | `relative_path`、变量名 |
| `open_data_nsidc_smap_sample` | NSIDC/SMAP 下载→解压→MAT | Earthdata 凭证 | `relative_path` |
| `open_data_nasa_earthdata_sample` | Earthdata 下载→解压→提取 | Earthdata 凭证 | `relative_path`、变量名 |
| `open_data_esa_product_sample` | 欧空局包下载→SAFE 解压 | Copernicus 凭证 | `relative_path` |
| `smap_soil_moisture_local` | **本地硬盘** SMAP 路径流水线 | 本机已有数据 | `data/source` 路径 |

种子文件位于仓库 `Code/backend/workflow_seeds/system/`，服务启动时会拷贝到运行目录（已存在则不覆盖）。

---

## 4. 凭证怎么配

1. 打开前端 **系统设置 → 数据源**
2. 「开放门户凭证」中填写 Earthdata / NSIDC / Copernicus 的 token 或用户名密码
3. 或设置环境变量：`BACKEND_EARTHDATA_TOKEN`、`BACKEND_NSIDC_TOKEN`、`BACKEND_COPERNICUS_TOKEN`（**数据库配置优先**）
4. 画布节点 `cred_profile` 填对应 id
