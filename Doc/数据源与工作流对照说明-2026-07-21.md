## 一、整体说明

本系统是一套**地理信息数据分析平台**：把多源遥感/气象/站点数据接入到统一目录，通过工作流（算法流水线）加工成专题产品，再以地图图层或叠加图的形式展示在前端。

整体数据流向可以这样理解：

```
[原始数据源]                       [工作流/算法模块]                    [产出产品]
卫星 HDF / 网格 NetCDF  ──►  daily_bundle / smap_daily      ──►  日尺度 .mat
   +                         / ndvi_daily / fy_daily              时间序列 .mat
站点观测 TXT/CSV  ─────────►  / inversion_daily                   GeoTIFF 栅格
静态地理 SHP/TIF  ─────────►  / block_inversion / omega_block      PNG 预览图
                              / station_daily / statistics
                              / fitting / export
                                       │
                                       ▼
                              前端地图图层 / 信息面板 / 时间轴
```

**当前研究区域**：中国区域（经纬度范围 73°E–137°E, 15°N–59°N）  
**目标对齐网格**：0.25° WGS84（176 行 × 256 列），与 ERA5 / SMAP 共格

## 三、工作流与算法模块对照表

### 3.1 已注册的算法模块

下列 14 个算法模块已在系统中注册（通过 `@register_module_decorator` 装饰器），可在 LiteGraph 画布上拖拽组合成工作流：

| 模块名 | 算法目的 | 主要输入数据集 | 输出产品 |
|---|---|---|---|
| `smap_daily` | 将 SMAP L3 HDF5 文件转换为日尺度 MAT 产品 | SMAP_L3（HDF5） | 每日 `.mat`（含 SM/Ts/TBh/TBv/VWC/CF/lat/lon） |
| `ndvi_daily` | 8 天合成 NDVI → 日值插值 + Savitzky-Golay 滤波 | NDVI_VIIRS / NDVI_MODIS | 每日 NDVI `.tif` |
| `fy_daily` | 风云三号 MWRI 亮温产品日处理 | FY_MWRI_HDF | 每日亮温产品（多波段 HDF） |
| `station_daily` | 站点观测日均值 + 验证产物 | ISMN_STATION / CASMOS_STATION | 站点日值序列 `.mat`、验证报告 `.csv` |
| `daily_bundle` | 把多源数据装配成单日合成 MAT | SMAP + FY + NDVI + 土地覆盖 + 辅助场 | 单日 `.mat`（含 TB/SM/DH 等矩阵，作为反演输入） |
| `timeseries_bundle` | 把多个 daily_bundle 合成时间序列 MAT | daily_bundle 输出 | 时间序列 `.mat`（作为块反演输入） |
| `inversion_daily` | 单日 DDCA / 动态-H 反演土壤水分 | daily_bundle_mat | 单日反演 `.mat` |
| `block_inversion` | 8 天块批量反演（DH 或 DDCA 模式） | timeseries_bundle_mat + 可选 dh_mat | 块反演结果 `.mat` |
| `omega_block` | OMEGA 植被光学厚度反演（双温度 DUAL 方案） | timeseries_bundle_mat + omega_fixed_mat + exp0_calib_mat | OMEGA 时间序列 `.mat` |
| `statistics` | 统计分析（全局/分区/土地覆盖/时间序列） | 任意上游 manifest 或独立文件 | 统计图表 `.png` + 表格 `.csv` |
| `fitting` | 曲线拟合（线性/多项式/指数） | 任意上游 manifest 或独立文件 | 拟合曲线 `.png` + 参数 `.csv` |
| `export` | 通用导出（MAT/NetCDF/GeoTIFF/CSV） | 任意上游 manifest 或独立文件 | 多格式产品文件 |
| `remote_fetch` | 远程文件下载 + 物化（画布节点） | smb/sftp/ftp/gs/http URI | 本地缓存文件 |
| `archive_extract` | 压缩包解压（zip/tar） | `.zip` / `.tar` / `.tar.gz` | 解压后文件目录 |
| `config_read` | 读取配置文件（JSON/YAML/INI） | 配置文件 URI | 配置字典 |
| `variable_extract` | 从 MAT/HDF/NetCDF 提取指定变量 | 多格式文件 + 变量名 | 单变量数据 |
| `format_convert` | 格式转换（MAT ↔ NetCDF ↔ GeoTIFF ↔ CSV） | 多格式文件 | 目标格式文件 |
| `data_source` | 画布参数节点：声明数据源引用 | 任意数据集 URI | 数据源引用对象 |
| `output_map_layer` | 画布参数节点：声明地图图层输出 | 上游 manifest | 图层描述符 |

### 3.2 工作流预设（preset）

系统内置 1 个预设工作流：

| 工作流名 | 用途 | 流程 |
|---|---|---|
| `retrieval_workflow` | 反演主链：先建时间序列束，再跑块反演或 OMEGA 反演 | `timeseries_bundle` → `block_inversion`（DH/DDCA 模式）<br>或 `timeseries_bundle` → `omega_block`（OMEGA 模式） |

模式由 `algorithm_params["mode"]` 决定，可选值：`dh` / `ddca` / `omega`。

## 七、远程数据访问能力

### 7.1 支持的远程协议

| 协议 | URI 格式 | 用途 | 依赖库 |
|---|---|---|---|
| HTTP/HTTPS | `http(s)://host/path` | 在线开放数据 | requests |
| SMB | `smb://host/share/path?cred=profile_id` | Windows 共享 / NAS | smbprotocol |
| SFTP | `sftp://host/path?cred=profile_id` | SSH 文件传输 | paramiko |
| FTP / FTPS | `ftp(s)://host/path` | 传统 FTP | ftplib（明文 FTP 默认关闭） |
| Google Cloud Storage | `gs://bucket/path` | GCS 对象 | google-cloud-storage |
| MinIO / S3 | `s3://bucket/path` | 对象存储 | minio |
| 本地文件 | `file:///path` 或 `I:\...` | 本地磁盘 | pathlib |

### 7.2 远程数据接入方式

1. **设置页 → 远程存储**：配置 SMB/SFTP/FTP/GCS 凭证（profile_id 绑定 URI）
2. **图层注入**：通过环境变量 `BACKEND_REMOTE_LAYER_DATA_URIS` 或设置页 DB 覆盖，把远程 URI 注入到图层默认数据源候选列表的最前面，本地路径作为回退
3. **下载链**：`source_fetcher` 服务支持 http/https/minio/file/local 多协议抓取
4. **凭证库**：`workflow_state\remote_storage_credentials.sqlite3`，加密 key 与 GEE 共用

### 7.3 多机/Worker 部署提示

- 凭证**不**随任务载荷下发；Worker 进程需能打开同一凭证库路径并使用同一加密 key
- 推荐把凭证库父目录放到共享盘，或在每台 Worker 上重建相同 profile 后通过 `POST .../test` 验证
- API 与 Worker 都要能路由到 NAS（防火墙 / VLAN）

## 八、数据接入步骤（运维视角）

如果新接入一份数据，按以下步骤操作：

1. **下载/拷贝数据**到 `I:\Geograph_DataSet\{Category}\` 对应子目录（新数据集用英文目录名）
2. **注册数据集元信息**：在 `Code\algorithms\providers\Python\dataset_config.py` 的 `DATASET_REGISTRY` 字典中添加条目，声明 `name / logical_name / relative_path / file_format / variables / time_range / resolution / crs / tags`
3. **配置变量别名**（如使用非标准变量名）：在 `DailyBundleConfig` 等配置类中追加别名

5. **可选：配置 PNG 叠加**：在 `Code\backend\app\services\overlay_registry.py` 中添加 `OverlaySpec`
6. **重启后端**：`python launch.py restart fastapi`
7. **前端验证**：访问 `http://localhost:5173/`，在图层库中确认新图层

## 九、关键约束与注意事项

### 9.1 空间范围与投影

- **研究区**：中国区域，bbox = (73°E, 15°N, 137°E, 59°N)
- **目标对齐网格**：0.25° WGS84（与 ERA5/SMAP 共格，176 × 256）
- **全球栅格必须先裁剪到中国区域再投影 Web Mercator**，否则高纬度会出现拉伸伪影
- **EASE-Grid 2.0 9 km 真实像素大小是 9008.0552 m**（不是 9000.879 m，源自 .mat 元数据 500.4475 m × 18）
- **EPSG:3034 = LCC 兰伯特等角圆锥**（欧洲），**EPSG:3035 = LAEA 兰伯特方位等积**（欧洲），两者不可混淆
- **MapLibre image source 线性插值会导致 Web Mercator 投影下地理数据错位**，北方区域北偏约 2°

### 9.2 数据格式特殊点

- **SMAP HDF5** 变量名是 `latitude`/`longitude`（不是 `lat`/`lon`），系统已配置别名
- **SMAP 不提供 sand_fraction**，只有 clay_fraction
- **omega .mat 变量名是 `OMEGA_AVG`**（不是 `omega`）
- **CLCD 大文件** 228579×131361 uint8 = 28 GiB，转 float64 需 224 GiB，必须用 rasterio 窗口读取
- **ESACCI BIOMASS 大文件** 157500×405000 int16 = 1.28 TB，bool mask 需 3.28 GiB，必须分块读取
- **ERA5 DWAA/WDAA** 是事件标识层（0/1/255），不是连续 SMCI 值，366 波段对应 2020 闰年 366 天
- **彭守彰数据集单位**：降水 0.1 mm（值 ×0.1），温度 0.1 ℃（值 ×0.1）

### 9.3 远程访问约束

- **Cloudflare 隧道要求所有 HTTP 请求必须带 `User-Agent` 头**，否则返回 403 Forbidden
- **FileBrowser 用 `isDir` 字段判目录**（不是 `type` 字段，目录的 `type` 是空字符串）
- **远程数据只读**，不允许删除
- **Win11 E 盘有 4500+ 目录**，max-depth=6 扫描需 1.5+ 小时，建议用默认 max-depth=4

### 9.4 算法执行约束

- **块反演必须用 `ProcessPoolExecutor`**（不能用 `ThreadPoolExecutor`，GIL 会导致 0.52× 负加速）
- **并行进程必须用 spawn context**（兼容 Celery prefork）
- **自动 worker 数**：基于物理 CPU 核数（预留 2 核）+ 可用内存（200 MB/worker，2048 MB 系统预留）
- **并行失败自动回退到串行**（pickle/spawn/BrokenProcessPool 异常时）
- **Savitzky-Golay 滤波前必须填充边缘 NaN**（否则结果 NaN 传播）
