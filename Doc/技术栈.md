# 技术栈

说明：

- 本文件描述的是“目标架构 + 当前落地状态”
- 当前阶段进度请同时查看 `Doc/项目任务清单.md`

| 层级 | 技术选型 | 当前状态 | 说明 |
| ---- | ---- | ---- | ---- |
| **前端框架** | Vue 3 + TypeScript + Vite | 已落地 | 客户端层基础，当前前端主项目已实际采用该组合 |
| **3D 地球** | CesiumJS + vue-cesium | 未启动 | 仍属于中后期路线，当前项目已明确以 `2D-first` 为主 |
| **2D 地图** | MapLibre GL JS | 已落地 | 当前二维地图主栈，已完成主舞台、底图切换与交互演示 |
| **大数据叠加** | deck.gl | 规划中 | 目标用于百万级点/网格渲染，当前尚未进入实际集成 |
| **UI 与状态管理** | 自定义组件 + Pinia | 部分落地 | 当前未使用 `Naive UI`，而是采用自定义 Windy 风格组件体系与 `Pinia` |
| **API 框架** | FastAPI (Python) | 已落地 | 异步 API 框架，已作为后端统一入口 |
| **任务队列** | Celery + Redis | 骨架已落地 | 脚本、队列、worker 分流与调度策略已建立，仍需真实常驻联调 |
| **算法引擎** | Python + importlib | 已落地第一层 | 已支持 provider 动态加载，并已接入 `lab-output` |
| **GEE 集成** | ee Python API (服务端代理) | 规划中 | 尚未接入第一条真实代理链 |
| **关系数据库** | PostgreSQL + PostGIS | 规划中 | 当前状态存储先使用 SQLite，PostGIS 仍是后续基础设施任务 |
| **缓存层** | Redis | 部分落地 | 任务队列方向已建立，下载缓存元数据骨架已存在，尚未完整联调 |
| **瓦片服务** | Martin + TiTiler | 规划中 | 尚未进入真实切片服务接入 |
| **栅格存储** | MinIO (S3兼容) + 磁盘 | 部分落地 | 本地/MinIO 对象存储抽象已存在，正式联调未完成 |
| **API 网关** | Nginx | 未启动 | 仍属于部署阶段任务 |
| **容器化部署** | Docker + Docker Compose | 未启动 | 仍属于后续部署与运维阶段任务 |
