# 当前数据源、工作流与产出一览
 
> **依据**：系统当前图层目录与课题组数据说明（整理日期：2026-07-21）。  
> **怎么读**：先看「类别」，再看「用什么原始数据 → 走什么流程 → 得到什么结果」。  
> **说明**：部分数据是「演示/样例已就绪」，部分是「算法能跑但原始数据还需自行下载」；文中会单独标注。

---

## 一、先看懂三个概念

| 说法 | 通俗含义 |
|------|----------|
| **数据源** | 原始数据从哪里来（卫星、站点、气象网站、本地硬盘等） |
| **工作流 / 处理方式** | 系统怎么用这些数据（实时拉预报、算法清洗反演、或直接叠图展示） |
| **目标数据 / 产出** | 最后在地图上看到的产品，或算法写出的结果文件 |

**空间范围常见写法**

| 标注 | 含义（约） |
|------|------------|
| 全球 | 东西经 −180°～180°，南北纬 −90°～90° |
| 中国区域 | 约东经 73°～137°，北纬 15°～59° |
| 华南默认研究框 | 约东经 109.6°～117.4°，北纬 20.1°～25.6°（部分算法图层默认框） |

**时间范围**

- 有的是**实时/预报**（随当前时刻滚动）；
- 有的是**本地已下载的一段样例**（例如「2023 年 1 月 13 天」），不等于全历史库都已入库。

## 二、实时气象数据（在线拉、按地图视口出图）

这类数据**不靠课题组硬盘里的大文件**，而是天气服务按当前地图范围临时取数、切成瓦片显示。时间粒度多为**小时**。

### 2.1 可用的天气数据提供方

| 提供方（界面可选） | 通俗说明 | 分辨率 / 预报时长（按模型） | 覆盖 |
|--------------------|----------|------------------------------|------|
| Open-Meteo 在线 | 公网气象聚合服务 | 见下表各模型 | 全球（区域模型除外） |
| Open-Meteo 本地 | 本机部署的同类服务 | 与模型配置一致 | 同左 |
| WeatherAPI / OpenWeather | 商业气象接口（需配置密钥） | 随供应商 | 全球点查/网格能力视配置 |

**Open-Meteo 常用预报模型（白名单）**

| 模型 | 约分辨率 | 预报时效 | 覆盖 |
|------|----------|----------|------|
| ECMWF IFS | 0.25°（约 25 km） | 约 15 天 | 全球 |
| GFS | 0.25° | 约 16 天 | 全球 |
| ICON | 0.25° | 约 7.5 天 | 全球 |
| ICON-EU | 0.0625°（约 6～7 km） | 区域预报 | 欧洲 |
| JMA / Météo-France / GEM 等 | 服务端「无缝」拼接 | 约 4～11 天 | 全球为主 |

> **临时下载范围**：按用户当前地图窗口（瓦片 z/x/y）按需取数，不是一次性下载全球全时段。

### 2.2 气象图层：源 → 流程 → 产出

| 地图图层（产出展示） | 物理量 | 处理方式 | 备注 |
|----------------------|--------|----------|------|
| 风场（10m / 80m / 120m / 180m） | 风速风向 | 天气引擎网格预报 → 粒子/风羽等可视化 | 高度层对应近地面到边界层 |
| 风场（850 / 500 / 200 hPa） | 气压层风场 | 同上 | 约 1.5 km / 5.5 km / 12 km 高度参考 |
| 降水 | 小时降水 | 网格预报 → 填色图 | 单位 mm/h |
| 温度（2m / 80m / 120m / 180m） | 气温 | 网格预报 → 填色图 | ℃ |
| 气压 | 海平面气压 | 网格预报 → 填色/等压线 | hPa |
| 湿度 | 2 m 相对湿度 | 网格预报 → 填色图 | % |
| 能见度 | 地面能见度 | 网格预报 → 填色图 | m |
| 云量 | 总云量 | 网格预报 → 填色图 | % |
| 露点温度 | 2 m 露点 | 网格预报 → 填色图 | ℃ |

**产出形态**：地图上的动态气象图层（可跟时间轴），不是长期归档的「课题组成果库」；若需存档需另行导出。

## 三、课题组算法流水线（卫星 / 站点 → 计算 → 产品）

这类走「工作流」：选图层或画布节点 → 后端调度 Python 算法 → 生成结果再上图。  
默认研究框多为**华南框**（见上文），实际以任务参数为准。

### 3.1 一览表

| 界面名称 | 原始数据源 | 分辨率 / 规格 | 当前样例或所需时间范围 | 工作流（算法模块） | 目标产出 |
|----------|------------|---------------|------------------------|--------------------|----------|
| **植被指数 (NDVI)** | VIIRS NDVI（约 750 m）、MODIS NDVI（约 250 m），16 天合成 | 见左 | **原始数据待下载**后可跑 | `ndvi_daily`：合成日值化 + 滤波 | 日值 NDVI 栅格（如 GeoTIFF） |
| **遥感反演** | Sentinel-2（经 Google Earth Engine） | 导出尺度约 **10 m** | 按所选影像场景/日期 | GEE：读影像 → 算 NDVI → 导出清单 | NDVI 遥感指数产品 + 导出清单 |
| **SMAP 土壤湿度** | NASA SMAP L3（HDF5） | 约 **9 km**（EASE-Grid） | 本地样例约 **2023-01～2023-09，19 天**（有缺口填补说明） | `smap_daily`：读取/清洗/出图 | 土壤湿度、亮温、VOD 等；清洗后 HDF5/展示层 |
| **FY MWRI 亮温** | 风云三号 MWRI 多通道亮温 | 轨道/波段产品（通道 10～89 GHz V/H） | **原始 HDF 待下载**（FY-3B/3D） | `fy_daily` | 重投影亮温产品，地图亮温图层 |
| **站点土壤湿度** | ISMN（全球）、CASMOS（中国）站点 | 点位观测（非网格） | **站点文件待下载** | `station_daily` | 站点日值序列 / 时次矩阵；地图上为站点符号 |
| **单日反演** | 日打包 `.mat`（含生态/反演中间量） | 随输入网格（常与 9 km 产品配套） | 依赖本地 `daily_bundle` / DDCA 等 | `inversion_daily` | 单日 SM、VOD、诊断量 DH 等 |
| **块反演** | 时间序列打包 `.mat`（含 Omega SMAP/FY 反演包） | 同上 | 依赖本地时间序列包 | `block_inversion` | 多日 DH/SM/VOD/Tau 等矩阵产品 |
| **OMEGA 反演** | 同上时间序列包 | 同上 | 依赖本地时间序列包 | `omega_block` | OMEGA、SM_RET、VOD_RET、Tau_star 等 |
| **SMAP/Omega 交叉分析（实验）** | 实验样例链路 | — | 标注样例时段约 **2023-01** | 实验 Provider（联调用） | 演示用热力图；**非生产正式产品** |

### 3.2 算法产出文件类型（课题组侧）

| 产出类型 | 内容概要 | 常见格式 |
|----------|----------|----------|
| Omega 土壤水分 / 光学厚度相关 | 亮温反演得到的土壤水分与相关参数 | MAT、GeoTIFF |
| NDVI 日值化 | 16 天合成插成日值并滤波 | GeoTIFF |
| 站点日值 / 时次矩阵 | 验证与订正用 | MAT |
| SMAP 清洗产品 | 质量筛选后的 SMAP | HDF5 |
| FY 亮温产品 | 重投影后的 MWRI 亮温 | HDF |
| Block / 批量反演 | 多日块结果 | MAT |

## 五、画布「开放数据下载」预设（按需临时拉取）

工作流编辑器里可用「开放数据下载」节点，按**预设网站根地址 + 相对路径**临时下载，再解压/读变量/转格式。默认预设包括：

| 预设名 | 数据门户 | 典型用途 |
|--------|----------|----------|
| NOAA NOMADS | 美国 NCEP 数值产品门户 | 数值预报等开放文件 |
| NOAA GOES | 美国静止卫星影像 CDN | GOES 影像 |
| NASA Earthdata | NASA LP DAAC 云端数据 | MODIS 等地球科学数据 |
| ESA Copernicus | 欧空局哥白尼数据目录 | Sentinel 等 |

> **范围与分辨率**：由具体下载的产品决定；系统不会自动下载「全世界全部历史」，只下载节点里指定的路径文件，并可写入本地静态缓存（可按策略清理）。

还可配合：**远程网盘/NAS（SMB/SFTP 等）拉取 → 解压 → 读配置 → 提取变量 → 格式转换 → 挂到地图图层**。

## 八、阅读时请注意的三点

1. **「已就绪」≠「全球全历史都已下载」**  
   多数科研图层当前是**有限天数/月份的样例**，便于演示与联调；扩库需按同一目录规范继续放数据。

2. **「待下载」图层**  
   NDVI 原始合成、FY MWRI 原始轨道、ISMN/CASMOS 站点等：算法与界面已接好，但**本机缺原始文件时不能出正式结果**。

3. **实时天气 vs 课题产品**  
   - 实时天气：随视口临时取数，适合态势浏览；  
   - 课题产品：依赖本地库或 GEE/画布流水线，适合科研分析与出图存档。

## 九、若需补充新数据，请提供这些信息

给数据管理员时，尽量写清：

1. 数据名称与来源机构  
2. 文件格式（如 HDF、NetCDF、GeoTIFF、MAT、CSV）  
3. **空间范围**（全球 / 中国 / 研究区经纬度）  
4. **时间范围**（起止日期；是否只要样例）  
5. 物理量名称与单位  
6. **空间分辨率、时间分辨率**  
7. 是否有质量标记（如站点 `quality_flag`）
