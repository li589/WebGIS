# 工程决策纪要：配置、瓦片与契约

> 状态：**已锁定**（Phase 1 工程治理，2026-07-16）  
> 本文档记录已达成共识、不再反复争论的架构决策，供前后端与 CI 对齐。

---

## 1. 瓦片双入口（Tile Dual Entrance）

| 用途 | 正式入口 | 响应类型 |
|------|----------|----------|
| 天气图层（GeoJSON 矢量/风场等） | `GET /weather/tiles/{layer_id}/{z}/{x}/{y}` | `application/geo+json` |
| 天气点查 | `GET /weather/point` | JSON（`WeatherPointResponse`） |
| 图层可选天气源 | `GET /weather/providers-for-layer/{layer_id}` | JSON |
| 底图栅格（天地图/百度/OSM 等） | `GET /unified-tiles/{layer_id}/{z}/{x}/{y}` | `image/png` 等栅格 |
| 底图代理管理面 | `GET/POST /runtime/tiles/*` | JSON（缓存统计、清缓存、提供商列表） |

**决策要点：**

- 天气与底图**不得**混用同一 URL 前缀；`WeatherTileProvider` 不再注册到 `tile_provider_registry`。
- 旧 `/tiles/{provider}/...` 路由已移除；前端不得再引用。
- 管理面（`/runtime/tiles/*`）与数据面（`/unified-tiles`、`/weather/tiles`）职责分离。
- 天气出站统一经 `weatherengine.fetch_gateway`：支持 `provider` 钉源；`auto` 可按优先级 fallback；钉源失败不静默换源。
- 瓦片/点查缓存键含 `provider_id`，避免换源脏缓存。
- 商业源（WeatherAPI / OpenWeather）默认禁用，仅覆盖近地面四层；高度/气压层仍依赖 Open-Meteo。
- **格网几何（2026-07-21 锁定）**：格心 `(i+0.5)*resolution`；半开归属只约束格心；格元全球贴合、**不**裁进瓦片框。细节见 `Doc/天气渲染进度同步-2026-07-21.md` 与 `Doc/规范文档.md` §15.6。

## 2. 配置来源：环境变量 + 数据库

| 配置类别 | 写入入口 | 读取投影 |
|----------|----------|----------|
| 服务级开关 / 路径 / 队列 | `.env` → `Settings` | 启动时加载 |
| API Key（天地图、百度等） | `POST /config/api-keys*` | `effective_config` 合并 env + DB |
| 天气源 Provider | `POST /config/weather/providers*` | `weatherengine.provider_registry` + DB 覆盖 |
| GEE 凭证 | `POST /config/gee/accounts*` | `gee_credentials_repository` |
| 运行时调参（并发、TTL 等） | `PATCH /runtime/config` | `runtime_status_service` snapshot |

**决策要点：**

- 密钥明文**不**经 `/runtime/api-config` 下发；该 POST 端点返回 **410 Gone**。
- 单一配置投影由 `effective_config.hydrate_effective_config()` 在启动时完成。
- 前端设置页统一走 `/config/*`，不走已废弃的运行时密钥 POST。

---

## 3. 事件：诚实轮询（Honest Polling）

- 工作流事件接口：`GET /workflow-runs/{run_id}/events`（JSON 分页/增量轮询）
- **不使用 SSE / WebSocket** 作为当前正式契约；文档与前端实现均按轮询描述。
- 控制流接口保持轻量，适合高频轮询。

---

## 4. 目录权威（Catalog Authority）

- 图层目录唯一权威：`GET /layers`（`layer_catalog` + `run_readiness` 合并）
- 演示快照 `/demo/*` 默认关闭（`BACKEND_DEMO_ROUTES_ENABLED=false`），启用时仍带 `Deprecation` 头。
- 前端图层列表以 `/layers` 为准，不以 demo snapshot 为生产数据源。

---

## 5. 工作流 Bridge 与 Legacy 降级

- 正式调度链：`_BRIDGE_CHAIN`（gee → weather → algorithm → provider → weather_engine fallback）
- 无 bridge 匹配时**默认拒绝**（`ValueError`），不再静默落入 `analysis_workflow_service` / `download_workflow_service`。
- Legacy 处理器仅在 `BACKEND_LEGACY_WORKFLOW_HANDLERS_ENABLED=true` 时可用（开发/迁移用途）。

---

## 6. lab_output 样例契约

- 课题组算法产出（`lab_output`）通过 workflow 结果引用 + artifact 访问，不内联大 payload 到状态接口。
- `ResultKind.map_layer` 允许携带轻量 `render_hint` / `layer_assets` / `point_feature` 以加速地图渲染。
- 大对象（COG、GeoJSON 文件、表格）一律 artifact 引用 + 按需 GET。

---

## 7. 契约 CI

- 关键 OpenAPI 路径不得漂移：`/weather/tiles`、`/unified-tiles`、`/config`、`/workflow-runs`
- 检查脚本：`Code/backend/scripts/check_openapi_drift.py`
- 前端类型生成：`npm run gen:types`（基于 `Code/frontend/openapi.json`）
- 前端漂移检查：`npm run check:openapi`

---

## 8. 已废弃端点

| 端点 | 状态 | 替代 |
|------|------|------|
| `POST /runtime/api-config/{provider}` | 410 Gone | `/config/api-keys`、`/config/weather/providers` |
| `POST /frontend/commands` | 410 Gone | 前端本地处理；服务端用 `/workflow-runs` |
| `GET /demo/*` | 404（默认）或 Deprecation（启用时） | `/layers` + `/workflow-runs` |
