## 3. 当前项目情况

### 3.1 前端现状

状态：阶段性稳定

已完成：

- `2D-first demo` 主路径稳定
- `MapCanvas` 已支持地图主舞台、底图切换与广东边界数据接入
- `TimelineScrubber` 已支持连续时间轴演示
- `FloatingPanelFrame`、左右浮层、工具条已形成 Windy 风格的轻量化交互框架
- `demo-data / demo-adapter / demo-api / runtime-api` 已形成前端协议占位层
- **底图系统全面重构（2026-07-03）**：
  - `stores/ui.ts`：16 个瓦片源统一配置（含 `'none'` 无底图），ESRI/Google/Bing/OSM 等，含坐标系标注、色彩调整参数、是否标准源标记
  - `MapCanvas.vue`：移除旧的 `basemapMode` 逻辑，改用 `tileSourceId` + `showAdminOverlay` + `adminOverlayOpacity`
  - `ModeToolbar.vue`：替换旧的"行政区/OSM/混合"三段开关为"风格分类（影像/街道/深色/地形）+ 当前风格下的瓦片源按钮组 + 行政区边界开关"
  - 非标准瓦片源（天地图/百度/高德）标注 `needsBackendTransform: true`，前端不直接使用，仅为后端坐标转换预留接口
  - `DashboardView.vue`：适配新的 tileSourceId / showAdminOverlay props 接口
- **截图导出功能（2026-07-03）**：
  - `ScreenshotExport.vue`：新建截图导出组件，支持 4 种截图模式（带外壳/无外壳/无控件/纯净）× 2 种格式（PNG/PDF）
  - `ModeToolbar.vue`：新增截图按钮，通过 emit 触发截图面板
  - `DashboardView.vue`：通过 `mapCanvasRef` + `defineExpose` 获取地图舞台 DOM 引用并传递给截图组件
  - `html2canvas` + `jspdf` 依赖已安装
- **无底图模式（2026-07-03）**：
  - `stores/ui.ts`：`TileSourceId` 新增 `'none'` 瓦片源，`TILE_SOURCES` 数组首位新增无底图配置项
  - `MapCanvas.vue`：`switchTileSource` 拦截 `'none'` 直接隐藏瓦片层，仅显示叠加图层

未完成：

- 默认主界面尚未切换到真实后端数据主链
- 未接入 Cesium / deck.gl
- 未进入真实大规模数据渲染阶段
