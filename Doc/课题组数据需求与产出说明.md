# 课题组数据需求与产出说明

## 一、系统概述

本系统基于 `Code/algorithms/providers/Python` 中的算法包构建，提供土壤水分、植被指数、大气遥感等数据的处理与分析能力。

---

## 二、需要提供的数据（输入）

### 2.1 卫星遥感数据

| 数据集名称 | 格式 | 变量 | 分辨率 | 用途 |
|-----------|------|------|--------|------|
| VIIRS NDVI | HDF | NDVI | 750m | 植被指数监测 |
| MODIS NDVI | HDF | NDVI | 250m | 植被指数监测 |
| SMAP L3 土壤水分 | HDF5 | soil_moisture, TBh, TBv | 9km | 被动微波土壤水分 |
| FY-3 MWRI 亮温 | HDF | 10V, 10H, 18V, 18H, 23V, 36V, 36H, 89V, 89H | - | 国产微波成像仪 |
| MODIS 土地覆盖 | HDF | LC (IGBP分类) | 1km | 地表分类 |
| 融合降水数据 | GeoTIFF | precipitation | - | 降雨融合产品 |
| ECMWF 风场 | NetCDF | u10, v10, wind_speed | 0.25deg | 气象再分析 |
| GOSAT XCO2 | MAT | xco2 | 点位 | 大气CO2柱浓度 |

### 2.2 地面站点数据

| 数据集名称 | 格式 | 变量 | 用途 |
|-----------|------|------|------|
| ISMN 全球站点 | TXT/STM | soil_moisture, soil_temperature | 全球土壤水分验证 |
| CASMOS 中国站点 | TXT | soil_moisture, soil_temperature | 中国站点数据 |
| 中国生态站网 | TXT | soil_moisture, soil_temperature | CERN 长期观测 |

**站点数据格式要求**:
- 字段: year, month, day, hour, lat, lon, elev, depth_upper, depth_lower, soil_moisture, quality_flag, site_id, source
- quality_flag = 1 表示高质量数据

### 2.3 静态地理数据

| 数据集名称 | 格式 | 变量 | 用途 |
|-----------|------|------|------|
| 中国行政区划 | GeoJSON/SHP | boundary | 矢量边界 |
| SRTM DEM | GeoTIFF | elevation | 数字高程 (90m) |
| MODIS 辅助数据 | HDF | LST, albedo | 地表温度/反射率 |

### 2.4 算法专用数据字段（.mat 文件变量）

**Omega 算法需要的字段**（亮温反演土壤水分）:
```
TBv_mat / TBh_mat  - 亮温 (V/H 极化)
IA_mat              - 入射角
Ts_mat / TC_mat / TG_mat  - 地表/植被/土壤温度
SMref_mat          - 参考土壤水分
NDVI_mat           - 归一化植被指数
SF_mat             - 植被散射比例
Albedo             - 地表反照率
B                  - 参数 B
CF (clay_fraction) - 黏土含量
BD (bulk_density)  - 土壤容重
H / h              - 土壤湿度 (输出)
LC                 - 土地覆盖类型
omega_fixed_vec    - 固定粗糙度参数
omega_pft          - 基于植被类型的粗糙度
```

## 三、系统产出的数据（输出）

### 3.1 算法产出

| 产出类型 | 描述 | 格式 |
|---------|------|------|
| **Omega 土壤水分** | L波段主动/被动微波反演的土壤水分 (h, α, τ) | MAT / GeoTIFF |
| **NDVI 日值化** | 8天合成到日值插值 + Savitzky-Golay滤波 | GeoTIFF |
| **站点日值序列** | 站点土壤水分日均值时间序列 | MAT |
| **站点日值+时次矩阵** | 含日内时次变化的矩阵 | MAT |
| **SMAP 数据清洗** | 质量筛选后的 SMAP 产品 | HDF5 |
| **FY 亮温产品** | FY-3 卫星亮温重投影产品 | HDF |
| **Block 反演结果** | 8天块的批量反演结果 | MAT |

### 3.2 工作流输出结构

```
/output/
├── omega/                    # Omega 算法输出
│   ├── {region}_{date}.mat  # 土壤水分矩阵
│   └── {region}_{date}_block.mat
├── ndvi/                    # NDVI 处理输出
│   └── {region}_{date}_daily.tif
├── station/                 # 站点数据输出
│   └── {site_id}_{start}_{end}.mat
├── smap/                    # SMAP 清洗输出
│   └── {region}_{date}_cleaned.h5
└── fy/                      # FY 产品输出
    └── {orbit}_{date}.hdf
```

### 3.3 状态/元数据

| 类型 | 描述 |
|------|------|
| 运行状态 | idle / running / success / failed |
| 工作流摘要 | 算法参数、输入数据集、执行时间 |
| 数据可用性 | 时间范围、空间覆盖 |

## 五、课题组特有数据准备清单

如果需要添加课题组专有数据，需提供以下信息：

1. **数据来源**: 卫星型号 / 站点名称 / 实测数据
2. **文件格式**: HDF / NetCDF / GeoTIFF / MAT / CSV
3. **空间范围**: 研究区边界 (GeoJSON) 或经纬度范围
4. **时间范围**: 起止日期
5. **变量列表**: 包含的物理量
6. **分辨率**: 空间 / 时间分辨率
7. **质量控制**: 是否有质量标记字段

## 六、快速开始

1. 将数据放入 `I:\Geograph_DataSet\{对应子目录}`
2. 重启后端服务
3. 访问 http://localhost:5173/ 打开前端
4. 在图层库中添加相应图层进行可视化
