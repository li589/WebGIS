## 项目定位

**Comprehensive Geographic Data Analysis System** — 一个基于 Web 的综合地理信息分析平台，统一承载 3D 地球 / 2D 地图双模式展示、多源数据接入（本地 / GEE / 公共接口）、动态时空结果展示，以及多课题组算法模块化接入。目前已从"方案原型"进入"工程化落地"阶段。

## 整体架构

仓库分三层：`Doc/`（方案文档）、`Env/`（本地环境）、`Code/`（工程代码）。核心在 `Code/`，采用**前后端分离 + 协议先行 + 算法插件化 + 工作流驱动**：

```
Code/
├─ frontend/    Vue3 + TS + Vite + Pinia，WebGIS 展示壳层
├─ backend/     FastAPI 工作流服务（workflow-runs 主链）
├─ algorithms/  Python 算法包（contracts/workflow/data_access/modules/publish）
├─ shared/      前后端共享契约
├─ infra/       部署配置
└─ docs/        协作与事实同步文档
```

**核心运行模型是"控制流 + 数据流"双通道**：
- 控制流：`workflow-runs` 状态机、事件、取消、重试、运行态查询
- 数据流：结果视图、artifact 访问、`preview.png` 预览
- 结果统一返回 `WorkflowResultReference`，前端经 `result-adapter` 映射为图层/信息面板模型

**后端执行侧**并行存在多条链：`weatherengine`（fallback 天气产物）、`weather workflow DAG`（9 个天气节点：风/温/雨/湿/压/能见度/摘要等）、`python provider bridge`、`download real fetch`（http/minio/file/local）、`gee bridge`。已具备 `retry_pending` 中间态、`FailureCategory` 失败分类、自动重试语义。

**前端渲染架构**已确立统一原则：标准 GIS 数据 → MapLibre（GPU）；自定义叠加层（粒子流/风羽/等值线）→ Canvas 2D（CPU）。已完成 WebGL→Canvas 2D 迁移。
