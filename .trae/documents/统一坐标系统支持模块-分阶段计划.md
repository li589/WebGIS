# 统一坐标系统（CRS）支持模块 — 分阶段计划

## 背景

系统未来将接入多种地理数据源，坐标系涵盖：

- **国外**：WGS84、ETRS89（欧洲）、EASE-Grid 2.0 xxxkm
- **国内**：WGS84、CGCS2000、GCJ-02（POI 加密）、BD-09（POI 加密）
- **投影**：高斯-克吕格、UTM、兰伯特等角圆锥、Web Mercator（EPSG:3857）
- **附加**：部分数据在标准 CRS 基础上引入了额外偏移

**当前痛点（来自代码勘探）**：

1. [`coordinate_transform_service.py`](file:///d:/temp_desktop/Proj/Comprehensive%20Geographic%20Data%20Analysis%20system/Code/backend/app/services/coordinate_transform_service.py) 仅纯 Python 实现 GCJ-02/BD-09/EPSG:3857 四种转换
2. [`OverlaySpec`](file:///d:/temp_desktop/Proj/Comprehensive%20Geographic%20Data%20Analysis%20system/Code/backend/app/services/overlay_registry.py#L32-L83) **无 `crs` 字段**，bounds 隐式假设为 WGS84
3. [`/import/raster`](file:///d:/temp_desktop/Proj/Comprehensive%20Geographic%20Data%20Analysis%20system/Code/backend/app/api/routers/import_router.py#L47-L185) 把源 CRS 的 bounds 直接返回前端 → 非 WGS84 栅格在地图上错位
4. [`CsvImportDialog.vue`](file:///d:/temp_desktop/Proj/Comprehensive%20Geographic%20Data%20Analysis%20system/Code/frontend/src/components/toolbar/CsvImportDialog.vue#L29-L38) `CRS_OPTIONS` 仅 8 项，**缺 GCJ-02/BD-09/ETRS89/EASE-Grid/UTM 全系/Gauss-Krüger 全系/Lambert**
5. `meta.source_crs` 在 [`overlay-image-module.ts`](file:///d:/temp_desktop/Proj/Comprehensive%20Geographic%20Data%20Analysis%20system/Code/frontend/src/components/map/overlay-image-module.ts#L164) 中被读取但从未使用
6. EASE-Grid 9km 重投影逻辑散落在 [`Tools/export_overlay_assets.py`](file:///d:/temp_desktop/Proj/Comprehensive%20Geographic%20Data%20Analysis%20system/Tools/export_overlay_assets.py) 一次性脚本中，无可复用模块
7. 无自定义偏移补偿机制

**用户决策**（已确认）：
- 分阶段交付
- 自定义偏移 UX：显式 `lng_offset`/`lat_offset` 字段 + 双点校准（Phase 2）
- CSV/POI 转换：前端预览（proj4 + 纯 TS GCJ-02/BD-09）+ 后端最终转换（pyproj）
- 栅格 CRS：上传时检测 + 用户确认对话框
- **完成全部修改后进行代码审查（聚焦旧代码使用与逻辑混乱）→ 重启整个项目**

## 架构总览

三层设计，前后端镜像：

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: CRS Registry    — 声明式 CRS 目录（含分类、EPSG、proj4）  │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: CRS Transformer — pyproj 包装 + GCJ-02/BD-09 委托 + 偏移  │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: CRS Detector   — 从 rasterio/GeoJSON/.mat/.nc 自动检测   │
└─────────────────────────────────────────────────────────────┘
```

**关键设计**：
- GCJ-02/BD-09 不是标准 EPSG，作为 `CRSCategory.ENCRYPTED` 类别，用 `GCJ02`/`BD09` 简码（无 EPSG 前缀）
- 偏移在 CRS 转换**之后**应用（用户原话"在此基础上引入了偏移"）
- 栅格走两步流程：`POST /import/raster`（检测）→ `POST /import/raster/confirm`（确认+重投影）
- 旧 `coordinate_transform_service.py` 转为 deprecated 垫片（仅 2 个调用者，零修改）

## 备注

- pyproj 已随 rasterio 安装于本地 `Env/Python312/Lib/site-packages/`，无需新装

- 栅格走两步流程：上传检测 → 用户确认 → 触发重投影，避免静默错误转换
- 旧 `coordinate_transform_service.py` 转为 deprecated 垫片，仅 2 个调用者（`layer_router.py`、`tile_proxy_service.py`）零修改即可继续工作
