### 2.2 阶段 B：补齐前端测量功能

#### B.1 创建 `measure-module.ts`（交互模块）

**文件**：`Code/frontend/src/components/map/measure-module.ts`（新增）

**职责**：
- 绑定 MapLibre 事件：`click`（添加点）、`dblclick`（完成）、`contextmenu`（撤销）、`mousemove`（预览）
- 管理 3 个 MapLibre GeoJSON Source + Layer：
  - `measure-points` → `measure-points-layer`（圆点，半径 5px，白色边框 + 蓝色填充）
  - `measure-line` → `measure-line-layer`（实线，2px，蓝色 #2b7fff）
  - `measure-preview` → `measure-preview-layer`（虚线，2px，半透明蓝色）
- 协调 `MeasureCanvas` 标注层（show/hide/updateState）
- 模式切换时禁用 `doubleClickZoom`、`boxZoom`（dragPan 由 `mapInteractionModule` 管理）

**接口**：

```typescript
import type { Map as MaplibreMap } from 'maplibre-gl'
import { MeasureCanvas } from './measure-canvas'
import type { InteractionMode, MeasurePoint, MeasureState } from '../../stores/ui'

export interface MeasureModule {
  bindEvents: () => void
  applyMeasureMode: () => void
  dispose: () => void
}

export function createMeasureModule(options: {
  map: MaplibreMap
  getInteractionMode: () => InteractionMode
  getMeasureState: () => MeasureState
  addMeasurePoint: (p: MeasurePoint) => void
  undoLastMeasurePoint: () => void
  completeMeasure: () => void
  setHoverPoint: (p: MeasurePoint | null) => void
  clearMeasure: () => void
}): MeasureModule
```

**关键实现要点**：

1. **GeoJSON 同步**：通过 `updateGeoJSON()` 函数从 `getMeasureState()` 读取最新 points + hoverPoint，重建 3 个 source 的 GeoJSON 数据。在每次 store action 后调用（click/dblclick/contextmenu/mousemove）。

2. **事件处理**：

```typescript
function onClick(e: MapMouseEvent) {
  if (options.getInteractionMode() !== 'measure') return
  options.addMeasurePoint({ lng: e.lngLat.lng, lat: e.lngLat.lat })
  syncAll()
}

function onDblClick(e: MapMouseEvent) {
  if (options.getInteractionMode() !== 'measure') return
  e.preventDefault()  // 阻止默认双击缩放
  options.completeMeasure()
  syncAll()
}

function onContextMenu(e: MapMouseEvent) {
  if (options.getInteractionMode() !== 'measure') return
  e.preventDefault()  // 阻止右键菜单
  options.undoLastMeasurePoint()
  syncAll()
}

function onMouseMove(e: MapMouseEvent) {
  if (options.getInteractionMode() !== 'measure') return
  if (!options.getMeasureState().isDrawing) return
  options.setHoverPoint({ lng: e.lngLat.lng, lat: e.lngLat.lat })
  syncAll()
}
```

3. **applyMeasureMode**：

```typescript
function applyMeasureMode() {
  const isMeasure = options.getInteractionMode() === 'measure'
  if (isMeasure) {
    options.map.doubleClickZoom.disable()
    options.map.boxZoom.disable()
    canvas.show()
    syncAll()  // 重绘已有路径
  } else {
    options.map.doubleClickZoom.enable()
    options.map.boxZoom.enable()
    canvas.hide()
  }
}
```

4. **图层添加时机**：在 `bindEvents()` 中检查 `map.getSource('measure-points')` 是否存在；若不存在，则在 `map.on('load', ...)` 或直接添加（若 map 已 loaded）。使用 `map.loaded()` 判断。

5. **dispose**：移除 3 个 source + layer，移除事件监听，调用 `canvas.dispose()`。

## 四、假设与决策

### 4.1 假设

1. **SMAP .mat 的 lat/lon 含 NaN 但不全为 NaN**：`_bounds_from_centers()` 的 `np.isfinite()` 过滤后应能保留有效坐标。若全为 NaN，需进一步排查 .mat 结构（备选：fallback 到 `_CHINA_BBOX`）。
2. **前端 `npm run typecheck` 存在**：若实际为 `npm run build` 或 `vue-tsc --noEmit`，需调整命令。
3. **后端启动脚本位置**：待确认 `Code/backend/run_start.bat` 或类似入口。
4. **`map-canvas-state.ts` 的 resources 类型**：若为强类型需扩展，若为宽松类型则跳过 B.6。

### 4.2 决策

1. **重跑整个脚本而非单独跑 SMAP**：简化实现，额外耗时仅 4-5 分钟，且能验证全量图层。
2. **measure 模式下禁用 dragPan**：与 select 模式行为一致，避免点击/拖动冲突。用户可切回 move 模式平移。
3. **measure 模式禁用 doubleClickZoom + boxZoom**：双击用于完成路径，框选缩放会干扰。
4. **路径线 + 圆点用 MapLibre GeoJSON Layer**：与底图同步缩放/平移，无需手动重绘。
5. **标注层（文字 + 预览虚线）用 Canvas 2D**：复用 `wind-particle-canvas.ts` 的 DPI 适配 + rAF 节流模式，文字描边效果 MapLibre 难以实现。
6. **缩放自适应阈值 60px**：段在屏幕上的投影长度低于 60px 时隐藏标签，避免密集标点时文字重叠。
7. **清除按钮条件显示**：仅在 measure 模式且有路径点时显示，避免误触。
8. **commit message 用中英混合**：遵循项目现有提交风格（project_memory + git-commit-message.md 规则）。

## 五、风险与回退

### 5.1 风险

1. **SMAP .mat 结构异常**：若 lat/lon 不在标准键名下（`lat`/`latitude`/`lon`/`longitude`），`_bounds_from_centers()` 仍会返回 NaN。**缓解**：A.2 验证失败时，用 Python REPL 检查 `list(m.keys())`，找到实际坐标键名后扩展 SMAP 读取逻辑。
2. **MapLibre image source 不支持 setCoordinates**：切换时间序列时只能 `setUrl`，bounds 不能动态变化。**当前实现**：bounds 在 `_addOverlay` 时一次性读取并缓存，时间序列各帧共享同一 bounds（合理，因为同一图层的网格不随时间变化）。
3. **measure-module.ts 的 GeoJSON source 添加时机**：若 map 未 loaded 时调用 `addSource` 会失败。**缓解**：在 `bindEvents()` 中检查 `map.loaded()`，未 loaded 则注册 `map.on('load', addSources)` 一次性回调。
4. **renderWorldCopies=true 跨 ±180° 经线**：测量路径跨越日期变更线时，`map.project()` 可能返回屏幕坐标跳跃。**缓解**：暂不特殊处理，路径若跨越 ±180° 视觉上会出现绕地球一周的连线（罕见场景，可后续优化）。
5. **前端 typecheck 连锁错误**：`map-interaction-module.ts` 的类型扩展可能引发 `MapCanvas.vue` 等调用点的类型不匹配。**缓解**：先 typecheck 单文件，逐步修复。

### 5.2 回退

- 后端修复已落盘且 14 个图层验证通过，无需回退；若 SMAP 重跑后仍为 NaN，可临时将 SMAP bounds fallback 为 `_CHINA_BBOX`（修改 SMAP 函数的 else 分支）。
- 前端测量功能为新增模块，不影响现有功能；若编译失败可回退 B.2-B.6 的修改。
- Git 提交前确保所有验证通过；提交后若发现问题，可 `git revert` 单次提交。
